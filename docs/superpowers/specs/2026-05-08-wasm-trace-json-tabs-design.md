# Wasm Trace + JSON Tabs Design

## Goal

Change the Emscripten/XED web UI so a wasm run shows the HTML execution trace by default while still exposing the raw JSON result in a separate tab.

## Approved UX

Use two tabs after each analysis:

- **Trace**: default selected tab, displays Rust-generated execution trace HTML.
- **JSON**: displays formatted `UicaResult` JSON for debugging/copying.

No graph tab in this first pass.

## Architecture

### Rust API

Extend the `uica-emscripten` Rust API without changing the pure `uica-wasm` crate.

Current flow returns only `UicaResult` JSON. New flow returns a web envelope JSON string:

```json
{
  "schema_version": "uica-web-result-v1",
  "engine": "rust-emscripten-xed",
  "result": { "schema_version": "uica-result-v1" },
  "trace_html": "<!DOCTYPE html>..."
}
```

Errors continue to use the existing error envelope:

```json
{
  "schema_version": "uica-error-v1",
  "engine": "rust-emscripten-xed",
  "error": "..."
}
```

`run_request_json` will decode raw hex, verify the supplied UIPack, call `engine_output_with_uipack_runtime(..., include_reports=true)`, render the trace with `uica_core::report::render_trace_html`, and serialize the web envelope.

### JavaScript UI

`web/main.js` will parse the web envelope and populate two panels:

- Trace tab: set a sandboxed iframe `srcdoc` to `trace_html`.
- JSON tab: set `<pre>` text to formatted `result`.

The Analyze button still downloads/caches the selected `.uipack` exactly as today. The page should keep the existing cache status and throughput summary.

### Safety

The trace HTML is generated by our Rust code from decoded instruction data and report data. It should still be isolated in an iframe rather than injected into the parent DOM. Use a sandbox attribute that allows scripts inside the iframe because `traceTemplate.html` needs JavaScript for table behavior. The trace template must not load third-party CDN scripts; all trace behavior should be implemented with inline first-party JavaScript so trace data is not exposed to remote script origins.

### Compatibility

- Keep `rust/uica-wasm` pure wasm behavior unchanged.
- Keep CLI behavior unchanged.
- Keep shared manifest-selected `.uipack` data path unchanged.
- Do not add `instructions.json` or `instructions_full.json` fallbacks.

## Tests

Rust:

- Add `uica-emscripten` tests that assert successful run returns `uica-web-result-v1`, includes `result.schema_version = "uica-result-v1"`, and `trace_html` contains `Execution Trace` plus `var tableData = [`.
- Keep invalid hex and UIPack arch mismatch error tests.

Web/build:

- Shell syntax checks still pass.
- Existing web build path should still produce `dist/index.html`, `dist/emscripten/uica_emscripten.js`, and shared data.

Manual smoke:

1. Run `./scripts/build-web.sh` with emsdk active.
2. Serve `dist/` locally.
3. Open `/`.
4. Analyze `48 01 d8` on `SKL`.
5. Confirm Trace tab shows the execution trace iframe.
6. Confirm JSON tab shows formatted `uica-result-v1` JSON.

## Non-goals

- No graph tab in this pass.
- No object-file upload.
- No change to pure wasm decoded-only page.
